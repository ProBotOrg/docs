# VIP Защита: Поручите ProBot обеспечить безупречную защиту вашего сервера

:::caution 

VIP Защита является премиум-функцией и доступна только премиум-подписчикам. Если у вас нет премиума, вы должны перейти на него, чтобы получить доступ к этой особенности.

:::

ProBot — это мощный бот для Discord, который поможет вам в управлении сервером и обеспечит разными функциями, чтобы улучшить опыт сообщества. Одной из таких функций является VIP Защита, которая обеспечивает безопасную среду для вашего сервера, ограничивая действия пользователей с правами и ограничивая их возможность выполнять определенные команды.

:::info

Панель управления — это место, где осуществляется управление описанными здесь конфигурациями.

1. Перейдите в панель управления и авторизуйтесь. 
2. Выберите ваш сервер в левом меню.
3. Выберите модуль и убедитесь, что он включён. 

:::

## Добавление особых ролей

По умолчанию роль `@everyone` не имеет лимитов. Тем не менее, ProBot позволяет вам добавить любую желаемую роль и установить ограничения и лимиты.

## Лимиты прав

С ProBot вы можете дополнительно ограничить действия пользователей с правами и предотвратить возникновение проблем в будущем. VIP Защита позволяет установить ограничения на следующие действия:

- Бан
- Кик
- Каналы (Это ограничение будет применяться при создании/обновлении/удалении) 
- Роли (Это ограничение будет применяться при создании/обновлении/удалении)

:::note 

Обратите внимание, что лимит ролей не включает обновления ролей пользователя.

:::

## Действия

VIP Защита предоставляет три различных действия для каждого лимита прав:

1. Удалить административные разрешения со всех ролей. 
2. Удалить административные разрешения с ролей пользователя (**РЕКОМЕНДУЕТСЯ**). 
3. Удалить все административные роли с участника. 

Вы можете выбрать действие, которое лучше всего подходит для вашего сообщества.

:::caution

При выборе первых двух вариантов владелец сервера получит в ЛС резервный код для восстановления прав роли.

:::

## Лимиты команд

VIP Защита также предоставляет ограничения на определённые команды, которые вы можете установить с помощью команд ProBot. Следующие команды могут иметь установленные ограничения:

- Мут
- Голосовой мут
- Тайм-аут
- Кик
- Бан
- Роль

Для команды `/role` у вас есть две дополнительные опции:

- **Отключённые роли:** Препятствует пользователям с разрешением `/role` от предоставления определённых ролей, которые они могут выдать.

- **Разрешённые роли:** Разрешает пользователям с разрешением `/role` выдавать определённые роли.

:::caution

Пользователи с административными правами обходят опции ролей

:::

:::tip

Если у участника несколько ролей, то будут применены более высокие лимиты. Роль по умолчанию для всех – @everyone.

:::

:::warning

Эти лимиты не применяются к ботам из-за их работы, поэтому будьте осторожны, приглашая неизвестных ботов.

:::

## Лимиты

Обратите внимание, что количество доступных вариантов не ограничено, поэтому смело выбирайте желаемое количество без каких-либо ограничений.

## Опции защиты

VIP Защита предоставляет дополнительные возможности для обеспечения безопасности вашего сервера:

### Роли без административных прав

ProBot запретит этим ролям иметь административные разрешения, удалив их, когда они будут предоставлены.

:::tip

Рекомендуется добавить `@everyone` и `автороль` в этот список

:::

### Действие, когда другой бот выдал административную роль

ProBot может обнаружить, когда другой бот предоставил административные права, и предпринять следующие действия:

1. Удалить все административные роли с бота (**РЕКОМЕНДУЕТСЯ**). 

2. Удалить все административные права со всех ролей (**Уровень 2**).

3. Кикает бота.

Когда вы выберете действие, появится новая опция **Белый список ботов**. Вы можете ввести определённые ID ботов, которые, по вашему мнению, безопасны для использования и безопасны для вашего сообщества. Добавив бота в белый список, ProBot будет обходить его и не предпринимать никаких действий.

### Кто может получить доступ к панели управления

По умолчанию любой пользователь с правом `Администратор` имеет доступ к панели управления VIP Защитой. Однако, вы можете ограничить доступ к панели управления указанным пользователям. Доступны 2 варианта:

1. Любой пользователь с правом `Администратор`

2. Владелец + Указанные пользователи с правом `Администратор`

Когда выбрано **2**, появится поле, где вы можете ввести ID пользователей, также имейте в виду, что эти пользователи должны иметь право `Администратор`.

Напоследок, пользователи с административными правами по умолчанию имеют доступ к защите, в которой они смогут редактировать модуль защиты по своему усмотрению без ограничений. Однако у вас будет возможность ограничить доступ к модулю защиты всем пользователям, независимо от того, какое разрешение они имеют.
